\doxysubsection{Detailed Description}
Base Mono\+Behaviour that drives execution of a \doxylink{class_feazeyu_1_1_r_p_g_systems_1_1_dialogue_1_1_graph_asset}{Graph\+Asset}. 

Responsibilities:\+ • Builds the runtime blackboard once (reused across runs so non-\/shared state persists; Shared variables are global via Shared\+Blackboard\+Store) • Walks the graph along edges using independent {\bfseries{flow tokens}} • Dispatches each node to a registered \doxylink{interface_feazeyu_1_1_r_p_g_systems_1_1_dialogue_1_1_i_graph_node_handler}{IGraph\+Node\+Handler} • Handles structural built-\/in nodes:\+ Start, End, Condition, Set\+Variable, Run\+Subgraph (no system-\/specific knowledge)

── Parallel flow ──────────────────────────────────────────────────── Execution is carried by one or more Flow\+Tokens, each an independent cursor with its own current node and context. An output port wired to several nodes {\bfseries{auto-\/forks}}:\+ one token per edge. A token ends when its node dead-\/ends or hits an End node; the graph ends when the last token finishes or a handler calls {\ttfamily ctx.\+End()} (the terminal path used by quest Complete/\+\+Fail nodes). Single-\/edge graphs (all existing dialogue/\+quest content) run as exactly one token, so behaviour is unchanged.

To add a new graph-\/based system (quest, cutscene, …):\+
\begin{DoxyEnumerate}
\item Subclass \doxylink{class_feazeyu_1_1_r_p_g_systems_1_1_dialogue_1_1_graph_runner}{Graph\+Runner}.
\item In \doxylink{class_feazeyu_1_1_r_p_g_systems_1_1_dialogue_1_1_graph_runner_a0883a615c8adc62c6d5c42cd579f9911}{Awake()} call \doxylink{class_feazeyu_1_1_r_p_g_systems_1_1_dialogue_1_1_graph_runner_a7996fdb2293cb06f7df139785acdb4a0}{Register\+Handler()} for each node type you own.
\item Alternatively, place \doxylink{class_feazeyu_1_1_r_p_g_systems_1_1_dialogue_1_1_graph_node_behaviour}{Graph\+Node\+Behaviour} subclasses on the same Game\+Object — they are discovered automatically.
\end{DoxyEnumerate}

Node types without a registered handler are skipped with a warning, following the "{}\+Out"{} port if one exists.
