\doxysubsection{Detailed Description}
A lightweight alternative to Quest\+Graph\+Asset. 

Use this when a quest doesn\textquotesingle{}t need graph-\/based logic — no branching objectives, no conditional flow, just "{}player did the thing → quest is complete."{} The simple quest is a plain Scriptable\+Object with a title, description, and a boolean state; game code calls Complete (or Fail) when the condition is satisfied.

Also usable as a blackboard value, so chain graphs can reference simple quests the same way they reference graph quests. When a chain encounters a \doxylink{namespace_feazeyu_1_1_r_p_g_systems_1_1_quest}{Quest} Reference linked to a Quest\+Asset (rather than a Quest\+Graph\+Asset), no Quest\+Runner is spawned — completion is driven externally by the game calling Complete, which the chain runner observes via Quest\+Chain\+Runner.\+Notify\+External\+Quest\+Completed.

State lives on the Scriptable\+Object (it is the shared, inspector-\/wired identity that both the chain runner and external game code mutate, so it cannot be cloned per-\/runner the way a Blackboard is). Because mutating a serialized asset during play leaves the in-\/memory copy dirty after exiting play mode, the runtime state (m\+\_\+\+State) is reset to Quest\+State.\+Not\+Started automatically at the start of every play session — see Reset\+Runtime\+State\+On\+Play and On\+Enable — so editor runs always begin from a clean slate without any explicit game-\/start cleanup. Reset remains available for manually re-\/arming a quest mid-\/session.
