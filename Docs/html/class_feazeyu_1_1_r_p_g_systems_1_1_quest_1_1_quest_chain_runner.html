Detailed Description

Runtime driver for a QuestKind.Chain quest graph.

The chain is a static dependency description, not a linear flow: every RunSubgraph node represents a quest; edges mean "prerequisite". The runner maintains completion state and exposes the topological frontier as the set of Available quests.

── Supported quest kinds ──────────────────────────────────── Each Quest Reference node's "Quest" field must be linked to a blackboard variable. The variable type decides how the quest is driven:

  • Variable of type QuestGraph holding a QuestGraphAsset: StartQuest spawns a child GameObject with a QuestRunner that walks the quest's graph, and the chain observes its OnQuestEnded event.
  • Variable of type Quest holding a QuestAsset (simple quest, no graph): StartQuest does not spawn a runner — it calls QuestAsset.Start on the asset and waits for external code to call NotifyExternalQuestCompleted (or NotifyExternalQuestFailed) when the player's condition is satisfied.

Supporting both lets chains mix graph-based quests (with objectives and rewards) and simple flag quests without forcing every quest to be a full graph.
