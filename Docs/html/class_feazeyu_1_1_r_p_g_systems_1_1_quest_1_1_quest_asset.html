Detailed Description

A lightweight alternative to QuestGraphAsset.

Use this when a quest doesn't need graph-based logic — no branching objectives, no conditional flow, just "player did the thing → quest is complete." The simple quest is a plain ScriptableObject with a title, description, and a boolean state; game code calls Complete (or Fail) when the condition is satisfied.

Also usable as a blackboard value, so chain graphs can reference simple quests the same way they reference graph quests. When a chain encounters a Quest Reference linked to a QuestAsset (rather than a QuestGraphAsset), no QuestRunner is spawned — completion is driven externally by the game calling Complete, which the chain runner observes via QuestChainRunner.NotifyExternalQuestCompleted.

State lives on the ScriptableObject (it is the shared, inspector-wired identity that both the chain runner and external game code mutate, so it cannot be cloned per-runner the way a Blackboard is). Because mutating a serialized asset during play leaves the in-memory copy dirty after exiting play mode, the runtime state (m_State) is reset to QuestState.NotStarted automatically at the start of every play session — see ResetRuntimeStateOnPlay and OnEnable — so editor runs always begin from a clean slate without any explicit game-start cleanup. Reset remains available for manually re-arming a quest mid-session.
